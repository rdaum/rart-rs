Expand description

Core trait defining the interface for tree operations.

This trait provides the fundamental operations that can be performed on a tree data structure. It supports both generic key types (via Into<KeyType>) and direct key references for optimal performance.

§Design Philosophy

The trait provides two versions of most operations:

  • Generic methods (get, insert, remove) that accept any type convertible to KeyType
  • Direct methods (get_k, insert_k, remove_k) that work with key references

This allows for ergonomic usage while maintaining performance when needed.

§Examples

use rart::{AdaptiveRadixTree, ArrayKey, TreeTrait};

let mut tree = AdaptiveRadixTree::<ArrayKey<16>, i32>::new();

// Generic insertion (string converted to ArrayKey)
tree.insert("hello", 42);

// Direct key reference insertion (more efficient)
let key: ArrayKey<16> = "world".into();
tree.insert_k(&key, 84);

// Both work for queries too
assert_eq!(tree.get("hello"), Some(&42));
assert_eq!(tree.get_k(&key), Some(&84));
